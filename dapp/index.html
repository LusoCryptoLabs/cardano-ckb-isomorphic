01 - The thesis

A lock-and-mint bridge verifies both chains on both sides. Isomorphic binding doesn't.

An asset's ownership is anchored to one chain's UTxOs while its state and transition logic live on the other - and only the follower chain verifies the anchor. Verifying CKB's Eaglesong PoW inside a Plutus budget is the killer for a symmetric bridge. Binding sidesteps it by making each leg one-directional. We built both.

CARDANO → CKB the cheap, natural direction

Ownership is anchored to a Cardano seal UTxO; the asset's state lives on CKB. CKB verifies a Mithril certificate - a stake-based BLS threshold multisig - directly in CKB-VM. Feasible because CKB scripts run arbitrary RISC-V crypto.

CKB → CARDANO the hard, mirror direction

Ownership is anchored to a CKB cell; the asset's state lives on Cardano. Cardano verifies a Groth16 SNARK of CKB consensus - Eaglesong PoW + header-MMR + tx-inclusion - in Plutus, using Cardano's BLS12-381 builtins.

04 - Be honest

Trust model, and what isn't done.

What secures it
✓
Ownership / double-spend - the anchor chain's own consensus.
✓
Cross-chain soundness - Mithril honest-stake-majority (verified in-script), Groth16 soundness + trusted setup / CKB PoW honest-majority for the mirror leg.
✓
No bonded committee, no governor in the safety path.
What is NOT done
○
Mainnet deployment · third-party security audit · real trusted-setup ceremony.
○
Ecosystem / liquidity · the full production mint stack (caps, reorg, replay) is only partly deployed.
○
Multi-user scale - the relayer/prover serialize; a best-effort experiment, one flow at a time.
○
Not more post-quantum than the two chains' own signature layers.
