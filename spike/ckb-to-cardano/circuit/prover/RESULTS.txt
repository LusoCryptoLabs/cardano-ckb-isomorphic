STACK: arkworks Groth16 over BLS12-381 + ark-r1cs-std (R1CS gadgets). Rationale: the proof drops
DIRECTLY into the cardano_bound.ak Groth16 verifier we already built and MEASURED on-chain (~22.7%

GADGETS (ark-r1cs-std, BLS12-381 Fr; differential-tested vs the native CKB crates, in-circuit):
  eaglesong_gadget.rs  - faithful port of CKB's Eaglesong sponge (R1, the PoW): 16xUInt32 state;
     per round = GF(2) bit-matrix (xor) + rotation layer + constant injection + 8x(add,rotl8,rotl24,add)
     ARX; 43 rounds; rate-256 sponge, delimiter 0x06, big-endian absorb / little-endian squeeze.
     gadget output == native eaglesong on {empty, 32 B, 48 B}.   MEASURED:
        1 permutation        ~232,000 constraints
        PoW (48-byte msg)     464,192 constraints
  blake2b_gadget.rs    - Blake2b-256 with 16-byte personalization = CKB's ckbhash (reused by R2 MMR,
     R3 CBMT, R4): UInt64 state; G = addmany(64-bit) + xor + rotr; 12 rounds; param block sets
     digest=32 + personalization. gadget output == native blake2b-rs("ckb-default-hash") on
     {empty, "abc", 192 B}.   MEASURED:
        per 128-byte block    ~48,820 constraints
        192 B (RawHeader)     103,040 constraints

AdvanceCKBCert - the TRUSTLESS checkpoint (the chain root is earned by PoW work, not assumed)
------------------------------------------------------------------------------------------------
difficulty_gadget.rs (new trustless-work crypto), differential-tested vs native:
  difficulty_verify - enforce diff == floor((2^256-1)/target) via a 256x256->512 multiply-compare
     (diff*target <= MAX < (diff+1)*target). Valid diff passes; tampered diff+1 REJECTED. ~58,179 constraints.
  add256 / mul256 - enforced 256-bit add (w/ carry) and 256x256 multiply (witness-and-verify).

advance_ckbcert.ak (Cardano validator): a checkpoint UTxO {chain_root, total_difficulty}. ADVANCE
spends it with a Groth16 proof + new state; it verifies the proof, binds public inputs to
(old_root, old_diff, new_root, new_diff), enforces new_total_difficulty > old (HEAVIEST CHAIN, 32B
compare), and continuity. Compiles. So anyone can advance the root by proving more work; the bridge
never trusts an unproven checkpoint.
